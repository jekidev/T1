# Repository instructions

- GitHub er source of truth. Arbejd på feature branches; push små milepæle.
- Læs den relevante fil i `specs/active/` før ændringer.
- Ingen feature uden et Spec ID.
- Bevar mobilfunktion ved 360 px.
- AI må foreslå tekst, men aldrig overskrive manuskript uden accept.
- Generér ikke operationelle instruktioner til virkelig kriminalitet.
- Commit aldrig `.env`, `.personal-framework/`, tokens eller provider-nøgler.
- Provider- og GitHub-secrets må kun eksistere server-side.
- Ingen falske transskriptioner, embeddings eller RAG-resultater i produktion.
- Opdatér `project.index.json` ved arkitekturændringer.
- Kør `./scripts/validate.sh` før færdigmelding.
