# Agent Instructions

1. GitHub er source of truth.
2. Læs relevant fil i `specs/` før større ændringer.
3. Lav små, reviewable og reversible commits.
4. Commit aldrig secrets, `.env`, `.casecraft/` eller maskinspecifikke paths.
5. Bevar mobilfunktion ved 360 px.
6. AI må foreslå tekst, men ikke overskrive manuskriptet uden accept.
7. Operationelle instruktioner til virkelig kriminalitet må ikke genereres som del af produktfunktioner.
8. Ved større arbejde skal svaret struktureres som:
   - Plan
   - Changes
   - Validation
   - Risks
9. Kør `npm run validate` før færdigmelding.
