# Spec Checklist

Before marking a spec ready for implementation, confirm it has:

- [ ] Spec ID
- [ ] Status
- [ ] Problem Statement
- [ ] Goals
- [ ] Non-goals
- [ ] Functional Requirements
- [ ] Non-functional Requirements
- [ ] User Stories
- [ ] Technical Design
- [ ] Database Changes
- [ ] API Changes
- [ ] Security & Privacy Review
- [ ] Testing Strategy
- [ ] Rollback Plan
- [ ] Acceptance Criteria
- [ ] Future Improvements

Before merging the implementing PR, confirm:

- [ ] Docs updated if architecture changed
- [ ] `project.index.json` updated
- [ ] PR references the Spec ID
- [ ] Spec moved to `specs/completed/`
