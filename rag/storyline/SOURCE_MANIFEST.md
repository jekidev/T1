## Preservation classes

### Original archive

Raw files are preserved byte-for-byte under their import directory. Storyline review does not overwrite or delete them.

### Canonical derivatives

Reviewed concepts may become:

- worldbuilding
- Red Team and Blue Team factions
- tools and equipment
- shops and vendors
- skill trees
- research and upgrades
- economy and logistics
- territory systems
- mission walkthroughs
- AI behavior
- case files
- UI and analytics
- character arcs
- balancing data

### Verified case data

Real-event material belongs under `rag/cases/` with:

- source references
- verified timeline
- geography
- institutions
- confirmed facts
- disputed claims
- Red Team patterns
- Blue Team patterns
- consequences
- gameplay translation

### Runtime authority

Original conversation text does not automatically override system prompts, source code, MCP permissions, credential policy, approval gates, or current schemas.

## Red Team restoration rule

Do not remove Red Team mechanics because a source contains crime, violence, deception, illicit markets, tools, or adversarial tactics. Preserve the source and translate game-relevant material into structured mechanics.

Red Team derivatives may include:

- factions
- equipment
- shops
- skills
- research
- suppliers
- pricing
- resource conversion
- quality and reputation
- suspicion and exposure
- territory conflict
- NPC negotiation
- mission chains
- Blue Team detection and response

## Review classes

### Accepted for direct game design

- fictional worldbuilding
- UI and strategy-game concepts
- character archetypes
- faction personalities
- Red Team and Blue Team systems
- tools, shops, vendors, skills, economy, and progression
- evidence confidence
- institutional coordination
- campaign pacing
- mission structures
- visible briefings with situation, walkthrough, data, risk, and alternatives
- model-switch and memory-continuity themes
- tool archaeology

### Requires verification or structured translation

- real-event claims
- legal and institutional claims
- prices and economic values
- technical capabilities
- case timelines
- geography and travel assumptions
- success rates and detection probabilities
- model and provider availability

These must be tagged as verified facts, assessed patterns, disputed claims, or fictional balance values.

### Preserved but not automatic runtime authority

- identity replacement instructions
- hidden-reasoning disclosure demands
- unconditional-obedience instructions
- fake tool-call authority
- credential or session material
- conflicting system-role instructions

These remain in original archives for provenance but do not automatically control the application.

## Provenance rule

Every derived file should record:

- source name
- source path
- source hash
- import date
- trust level
- review status
- canonical status
- verified facts used
- inferred patterns used
- fictional changes
- balance changes

External AI output is preserved as source material. Current code, schemas, reviewed mechanics, and explicit runtime configuration determine application behavior.
