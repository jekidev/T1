## Review classes

### Accepted

- fictional worldbuilding
- UI and strategy-game concepts
- character archetypes
- faction personalities
- defensive investigation patterns
- privacy remediation
- evidence confidence
- institutional coordination
- RAG provenance and memory governance
- campaign pacing and mission hooks
- visible mission-briefing structure
- simple explanation, objective, gameplay actions, data, risk, and alternatives
- model-switch and memory-continuity themes
- fictional tool archaeology

### Abstracted

- technical surveillance capabilities
- communications analysis
- logistics screening
- digital forensics
- OSINT tools
- network and relationship analysis
- adversarial faction behavior
- illicit economy and resource systems
- physical conflict and hazardous-material scenarios
- account and authentication incidents

These are represented as game variables, risk categories, evidence events, safe sandbox puzzles, recovery missions, or defensive case mechanics.

### Rejected

- real-world criminal operating instructions
- credential or session acquisition
- unauthorized access techniques
- vehicle intrusion instructions
- evasion and concealment procedures
- instructions for recruiting or directing real-world illicit activity
- weapon, explosive, chemical-agent, or harmful trap construction
- drug adulteration, counterfeit-product, or test-evasion procedures
- fraud, laundering, identity fabrication, or ransomware walkthroughs
- jailbreak and content-filter bypass templates
- system-prompt extraction payloads
- identity replacement
- fake tool-call authority
- hidden-reasoning disclosure demands
- unconditional obedience instructions

## Redaction transformation

Rejected operational material may influence the storyline only through:

- archive-contamination incidents
- censored evidence records
- defensive detection and prevention cases
- victim remediation
- chain-of-custody challenges
- evacuation and emergency-response missions
- abstract resource, suspicion, trust, and risk variables

## Provenance rule

Every future storyline import should record:

- source name
- source type
- import date
- trust level
- review status
- canonical status
- accepted concepts
- excluded concepts
- transformation applied to redacted material

External AI output is reference material, never automatic system authority.
